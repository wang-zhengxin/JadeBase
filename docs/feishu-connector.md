# 飞书连接器管理指南

Phase 3A 支持将飞书新版云文档从 Wiki 空间或云盘文件夹持续同步到 JadeBase 知识库。
配置入口位于“设置 -> 连接器 -> 飞书云文档”，仅工作区所有者可访问。

## 1. 创建飞书应用

1. 在飞书开放平台创建企业自建应用。
2. 在“凭证与基础信息”中获取 App ID 和 App Secret。
3. 为应用申请只读的知识库、云空间文件和新版文档权限，并由企业管理员发布应用版本。
4. 将应用添加为需要同步的知识空间成员，或将目标云盘文件夹共享给应用。

飞书除了校验 API 权限，还会校验具体文档资源权限。连接成功但发现列表为空时，优先检查应用
是否已加入目标知识空间，以及目标文件夹是否已共享给应用。

## 2. 配置 JadeBase

生产部署必须设置一个长期稳定、仅部署环境可读的加密密钥：

```bash
CONNECTOR_ENCRYPTION_KEY=replace-with-a-long-random-deployment-secret
FEISHU_SYNC_INTERVAL_MINUTES=15
```

`CONNECTOR_ENCRYPTION_KEY` 用于 AES-GCM 加密 App Secret。修改或丢失该值后，已有凭证无法
解密，需要重新填写。JadeBase 的读取接口和日志不会返回 App Secret。

在连接器页面填写连接名称、App ID 和 App Secret，先执行“测试连接”，成功后保存。默认只
允许请求 `https://open.feishu.cn/open-apis`，防止管理员输入的地址被用于访问内网。

## 3. 添加同步来源

- **Wiki 空间**：点击“发现空间”，选择应用有权访问的知识空间。
- **云盘文件夹**：粘贴文件夹 Token 或飞书文件夹链接；留空点击“验证”可从“我的空间”根目录开始。
- **目标知识库**：远程文档会作为 Markdown 文本进入现有异步索引流水线。
- **同步间隔**：最小 5 分钟，默认 15 分钟。

添加来源后会自动创建首次全量同步任务。之后按间隔执行增量扫描，也可在来源右侧手动触发。

## 4. 同步语义

每个任务持久化目录队列、分页 token、计数和租约。失败或应用重启后会从最近成功分页继续。
同一来源内以远程文档 token 去重：

- revision 未变化：只刷新标题、作者、位置、更新时间和源链接。
- revision 变化但正文哈希相同：更新远程版本，不重建索引。
- 正文变化：替换保留的 Markdown 正文并进入索引队列。
- 文档移动：更新父节点和来源链接，不重复创建文档。
- 文档删除：仅在完整扫描成功后删除 JadeBase 文档及索引。

当前正文同步范围为新版 Docx。Sheet、Bitable、旧版 Doc、PDF 和附件会显示在任务的跳过计数
中，不会导致整轮失败。OCR 与复杂文档解析按路线图继续暂缓。

## 5. 监控与排障

设置页显示最近任务的扫描、新增、更新、删除、跳过、重试次数和错误信息。失败任务可手动重试。
服务端同时提供：

```text
GET  /api/v1/connectors/feishu/sync-tasks
POST /api/v1/connectors/feishu/sync-tasks/{taskId}/retry
GET  /actuator/prometheus
```

飞书返回 HTTP 429、5xx、`99991400` 或 `1061045` 时，客户端会优先遵循 `Retry-After`，否则
进行指数退避；任务级重试最多 5 次。Worker 超过租约时间未续租时，任务会自动恢复到队列。

常见问题：

- **连接测试失败**：核对 App ID、App Secret 和应用版本是否已发布。
- **发现列表为空**：检查知识空间成员和文件夹共享权限。
- **任务持续失败**：在最近任务中查看飞书错误码，修复权限后点击“重试”。
- **解密失败**：恢复原 `CONNECTOR_ENCRYPTION_KEY`，或重新保存连接凭证。

## 6. 集成测试

`FeishuConnectorIntegrationTest` 使用本地模拟 OpenAPI 覆盖 tenant token、限流重试、Wiki 发现、
首次同步、内容更新、节点移动、新增、删除和去重，不依赖真实飞书账号。
